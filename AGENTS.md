## 通用规范

- 使用 shadcn-ui 组件和变量，非必要不要自定义
- 每次回复开头必须加上："========== dai-shi.cn ============"
- 所有文案必须用中文
- 所有路由都用拼音（包括页面路径与页面内的锚点 ID）
- 不使用大量 CARD 风格
- 不可点击的元素不要添加 hover 效果或其他暗示可交互的视觉反馈
- 后续新增、删除或调整可访问页面路由时，必须同步更新 `llms.txt` 与 `sitemap.xml`

## 电话组件规范

- 所有与咨询、预约、电话相关的按钮或链接，必须使用 PhoneButton 或 PhoneLink 组件
- PhoneButton 和 PhoneLink 组件会自动使用配置在 site.ts 中的 SITE_HOTLINE_TEL 作为电话链接

## 二级页面规范

- 二级页面顶部返回条统一使用网站通用的 `PageTopNav` 组件
- 二级页面（除首页外）一律全屏布局：宽度由 `container mx-auto px-4` 控制，内容层不要再加 `max-w-Xxl` 二次约束；如需居中，使用 `text-center` 或 `mx-auto`

## 文案型/专题型详情页规范（参考 `/zhao-sheng-jian-zhang/2027`）

- 适用场景：服务流程、家长指南、专题说明、招生介绍、长文案详情等
- 整体气质：正式专题页/招生简章页风格，非博客或卡片列表页
- 首屏结构：大标题 + 导语 + 少量关键数据/目录摘要 + 明确 CTA，强调权威感、正式感、完整感
- 正文结构：左侧目录/侧边锚点 + 右侧主体内容的长页结构（短页面也需保留清晰目录感）
- 章节标题：做强层级设计，参考 `SectionHeading` 风格（章节标签、序号感、标题、说明文）
- 排版节奏：以留白、分组、浅底分区、边框分隔、图文穿插建立节奏，禁止连续卡片墙
- 卡片使用：仅少量用于承载资质点/环境点/承诺点/数据点，作为局部辅助模块
- 允许元素：浅灰背景分区、深色首屏 Hero、图片占位、数据指标、认证说明、分栏信息块
- CTA 位置：优先放在首屏和收尾，正文中少量插入
- 阅读节奏：大段落少而稳、章节边界清楚、视觉重心明确，避免碎片化资讯流或 FAQ 堆砌
- 结构化数据：所有可访问页面必须补充匹配的 JSON-LD（如 Article、FAQPage、Person 等）
